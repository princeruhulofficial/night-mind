# i18n Architecture

NightMind uses a lightweight, type-safe, zero-dependency internationalization system designed for a mobile-first React app.

## Goals

- **Type-safe keys** — TypeScript catches missing or misspelled keys at compile time.
- **Minimal boilerplate** — One hook (`useTranslation`) gives you everything.
- **Profile-driven language** — Language preference is stored in `profiles.language` and syncs across devices.
- **Graceful fallbacks** — Missing translations fall back to English. Missing provider falls back safely.
- **Interpolation** — Support for dynamic values: `t("welcomeName", { name: "Ruhul" })`.
- **No heavy libraries** — No i18next, no react-intl. Keeps bundle small.

## File Structure

```
src/
├── lib/i18n.ts                 # Dictionary + pure translate() function
├── hooks/useTranslation.tsx    # I18nProvider + useTranslation() hook
└── ...components that call useTranslation()
```

## How It Works

```
┌─────────────────┐
│  profiles.language │  ← stored in Supabase
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  useProfile()   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  I18nProvider   │  ← wraps the whole app (inside AuthProvider)
│  (context)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ useTranslation()│  ← used by any component
│  { t, lang }    │
└─────────────────┘
```

### Language Resolution Order

1. `profiles.language` (if user is logged in and has a preference)
2. Browser language (`navigator.language` starts with `bn` → Bangla)
3. Fallback → `"en"`

### Usage in Components

```tsx
import { useTranslation } from "@/hooks/useTranslation";

function MyComponent() {
  const { t, lang } = useTranslation();

  return (
    <div>
      <h1>{t("goodMorning")}</h1>
      <p>{t("welcomeName", { name: "Ruhul" })}</p>
      <span>Current language: {lang}</span>
    </div>
  );
}
```

### Changing Language

Language is changed by updating the profile:

```tsx
const update = useUpdateProfile();
await update.mutateAsync({ language: "bn" });
```

Because `I18nProvider` depends on `useProfile()`, the whole tree re-renders with the new language automatically (via React Query invalidation + realtime).

### Adding New Strings

1. Add the key to **both** `en` and `bn` objects in `src/lib/i18n.ts`.
2. Use it via `t("yourNewKey")`.

TypeScript will error if you try to use a key that doesn’t exist.

### Interpolation

```ts
// Dictionary
welcomeName: "Welcome, {name}"
// or
welcomeName: "স্বাগতম, {name}"

// Usage
t("welcomeName", { name: user.name })
```

Only `{word}` style placeholders are supported (no complex ICU for now).

### Non-React / Utility Code

For places outside React (e.g. `format.ts`):

```ts
import { translate, type Lang } from "@/lib/i18n";

export function greeting(lang: Lang = "en") {
  // ...
  return translate(lang, "goodMorning");
}
```

### Future Improvements

- Pluralization helper (`t_plural`)
- Date/time localization (`Intl.DateTimeFormat`)
- Lazy-loaded language packs if more languages are added
- AI system prompts that respect user language
- Extract remaining hardcoded strings from Tasks, Leaderboard, Onboarding

---

**Design principle**: Keep it simple until it hurts. This system is deliberately small so it stays understandable and fast.
