# Database Schema

All tables live in the `public` schema and are protected by Row Level Security (RLS).

## Core Tables

### `profiles`
Extends `auth.users`.

| Column          | Type          | Notes                                      |
|-----------------|---------------|--------------------------------------------|
| `id`            | uuid (PK)     | References `auth.users.id`                 |
| `name`          | text          | Display name                               |
| `sleep_time`    | text          | Format `HH:MM` (default `22:30`)           |
| `wake_time`     | text          | Format `HH:MM` (default `07:00`)           |
| `focus_areas`   | text[]        | e.g. `['health', 'work', 'learning']`      |
| `onboarded`     | boolean       | Whether the user finished onboarding       |
| `avatar_url`    | text          | Public URL from Storage                    |
| `birth_date`    | date          | Used by Life Clock                         |
| `language`      | text          | `'en'` or `'bn'`                           |
| `created_at`    | timestamptz   |                                            |
| `updated_at`    | timestamptz   |                                            |

A trigger (`handle_new_user`) automatically creates a profile row when a new user signs up.

### `tasks`
Daily plan items generated by the AI or created manually.

| Column             | Type              | Notes                                      |
|--------------------|-------------------|--------------------------------------------|
| `id`               | uuid (PK)         |                                            |
| `user_id`          | uuid              | Owner                                      |
| `title`            | text              |                                            |
| `description`      | text              | Optional notes                             |
| `icon`             | text              | Key for `TaskIconTile` (target, dumbbell…) |
| `duration_minutes` | int               | Default 30                                 |
| `priority`         | enum              | `high` / `medium` / `low`                  |
| `energy`           | enum              | `high` / `medium` / `low`                  |
| `status`           | enum              | `pending` / `done` / `skipped`             |
| `scheduled_for`    | date              | Which day this task belongs to             |
| `sort_order`       | int               | Display order                              |
| `starred`          | boolean           |                                            |
| `reminder_at`      | timestamptz       | Optional local notification time           |
| `completed_at`     | timestamptz       | When marked done                           |
| `created_at`       | timestamptz       |                                            |

### `checkins`
Stores night check-in / reflection conversations.

| Column       | Type        | Notes                                      |
|--------------|-------------|--------------------------------------------|
| `id`         | uuid (PK)   |                                            |
| `user_id`    | uuid        |                                            |
| `kind`       | text        | `'night'` or `'why'`                       |
| `messages`   | jsonb       | Conversation history                       |
| `summary`    | text        | AI-generated summary                       |
| `created_at` | timestamptz |                                            |

### `credibility_events`
Immutable log of score changes.

| Column       | Type        | Notes                                      |
|--------------|-------------|--------------------------------------------|
| `id`         | uuid (PK)   |                                            |
| `user_id`    | uuid        |                                            |
| `category`   | text        | e.g. `completed_tasks`, `honesty`          |
| `delta`      | int         | Positive or negative points                |
| `note`       | text        | Optional context                           |
| `created_at` | timestamptz |                                            |

### `patterns`
AI-detected behavioral patterns.

### `weekly_debriefs`
Generated weekly summaries (completion rate, top win, biggest blocker, AI suggestion).

### `why_answers`
Reasons users give when they skip a task.

## Key Functions

### `get_leaderboard()`
`SECURITY DEFINER` function that returns ranked users:

```ts
{
  user_id: string
  name: string
  avatar_url: string | null
  completed_week: number
  completed_total: number
  rank: number
}
```

Used by both the full Leaderboard screen and the “Top performers” card on Home.

## Storage

- Bucket: `avatars`
- Path convention: `{user_id}/avatar-*.{ext}`
- Public read, authenticated write restricted to own folder

## Realtime

The following tables are part of the `supabase_realtime` publication (with `REPLICA IDENTITY FULL` where needed):

- `tasks`
- `credibility_events`
- `profiles`
- (and related tables used by the live UI)

Clients subscribe via Supabase Realtime channels and invalidate TanStack Query caches on changes.
